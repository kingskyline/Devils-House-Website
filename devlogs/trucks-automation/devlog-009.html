Alpha v0.009 08/06/2026

UI & Systems Overhaul Update

King Skyline [DHG]

🎮 Player Update

Hey everyone! This development cycle was focused on one thing: making the game feel like a real game rather than a prototype.

Instead of adding new factories and roads, we spent time rebuilding the core systems that players interact with every second they play.

⸻ A Brand-New Settings System

The game now has a fully featured settings menu that saves your preferences and can be accessed from both the Main Menu and the Pause Menu. Settings include things like:

  • Auto-save controls
  • UI notifications
  • Vehicle fuel settings
  • Gameplay options
  • Future expansion settings

Everything syncs automatically and is built to expand as the game grows.

⸻ New In-Game HUD

The old placeholder UI has been replaced with a cleaner industrial-themed HUD. You can now quickly monitor:

  • Current Money
  • Income Per Minute
  • Fleet Capacity

The HUD updates instantly whenever something changes, making the factory feel much more responsive.

⸻ Time & Speed Controls

Managing a factory means managing time. The game now includes: Play, Pause, 2x Speed, 3x Speed, 4x Speed, and 5x Speed.

A new calendar system also tracks days, hours and minutes as your company grows.

⸻ Factory Analytics Dashboard

The Pause Menu has been completely redesigned. Instead of a simple menu, it now acts as a company dashboard showing:

  • Active Map
  • Total Play Time
  • Trucks Owned
  • Buildings Placed

Giving you a quick overview of your growing logistics empire.

⸻ New Loading Screens

Loading between maps is now much smoother. The game now loads factories in the background while displaying:

  • Animated loading screen
  • Progress indicator
  • Smooth transitions

No more sudden screen freezes or abrupt scene changes.

⸻ Current Status

This update was all about foundations. While there aren't any flashy new factories yet, the game now has a much stronger framework for future content, larger maps, and a much smoother player experience. The factory keeps growing!

⚙️ Technical Dev Log

Focus: UI Architecture, Time Systems & Scene Management

⸻ 1. Settings System Architecture

Implemented a centralized SettingsManager responsible for storing and synchronizing gameplay and UI preferences. Features include:

  • Auto-save configuration
  • Notification preferences
  • Fuel consumption settings
  • Time scale limits
  • Shared persistence architecture

The Settings Menu was converted into a reusable prefab, allowing identical functionality from both Main Menu and Pause Menu contexts.

⸻ 2. Event-Driven HUD Framework

Rebuilt the in-game HUD using the project's event-driven architecture. Components include Current Money, Income Per Minute, and Fleet Capacity.

EconomyManager now broadcasts state changes directly to subscribed UI elements, eliminating unnecessary polling and reducing UI overhead.

⸻ 3. TimeManager & Playback Controls

Implemented a dedicated TimeManager system featuring:

  • Day / Hour / Minute tracking
  • Pause state handling
  • Dynamic time scaling
  • Playback cycling (2x–5x)

Playback controls maintain previous speed states when pausing and correctly restore operation on resume.

⸻ 4. Pause Menu Analytics Framework

Expanded the Pause Menu into a data-driven analytics interface tracking:

  • Active map identification
  • Profile playtime
  • Active truck count
  • Total building count

Data is gathered dynamically from existing managers at pause time, ensuring accurate reporting without persistent UI updates.

⸻ 5. Asynchronous Scene Loading

Replaced legacy scene transition workflows with a dedicated LoadingScreenManager featuring:

  • Async scene loading
  • Non-blocking transitions
  • Radial progress visualization
  • Fade sequencing

Implemented a minimum display timer to prevent rapid scene flashes and maintain presentation consistency across varying hardware speeds.
