⚙️ Technical Dev Log

Focus: Developer Tooling, Multi-Map Architecture & Save System Evolution

⸻ 1. Developer Tooling Expansion

This development cycle focused heavily on creating custom editor tooling to improve workflow efficiency, reduce repetitive tasks, and accelerate balancing.

  • Economy Simulator: Created a dedicated Editor Window capable of simulating production chains and logistics networks. Calculates projected profit generation over time and automatically identifies bottlenecks caused by production limitations or transport capacity.
  • Building Blueprint Creator: Implemented a custom building generation workflow that automatically creates Prefabs, Data assets, Folder structures, and Asset references. Significantly reduces setup time.
  • Item Forge: Added a dedicated resource generation tool to create ResourceData assets with predefined values, establishing a standardized workflow.

⸻ 2. Runtime Debug & Monitoring Systems

  • Dev Console: Implemented a developer-only runtime console supporting currency injection, unlocking build menus, global level boosts, time acceleration, and mass upgrades.
  • Traffic Heatmap Overlay: Added road traffic monitoring where individual segments track vehicle usage. The overlay visualizes traffic density in real time to identify congestion points.
  • Fleet Inspector: Built a live vehicle management dashboard displaying truck state, current cargo, active status, and traffic conditions (includes emergency recovery tools).

⸻ 3. Core Engine Refactor – Multi-Map Support

This update represents one of the largest architectural changes made to the project.

  • CORE_GAME_MANAGER Architecture: Decoupled gameplay systems from individual maps. Economy, Grid, Input, Building and UI systems now operate independently as reusable architecture.
  • ProfileManager System: Implemented a persistent profile layer using DontDestroyOnLoad. Tracks global progression, playtime, and player data across scene transitions.
  • Loading Screen Framework: Added asynchronous scene loading support with fade transitions to prevent stuttering while large factory maps initialize.
  • Main Menu Framework: Added profile selection, multiple save slots, and map selection options.

⸻ 4. Save System Evolution

The save architecture was redesigned to support profile-based progression and multi-map gameplay.

  • Dynamic Save Structure: Save files are now generated dynamically using profile and map identifiers, supporting independent progression.
  • Resource Producer Progression: Added production-speed upgrade support to raw resource producers, fully integrated with save/load functionality.
  • Advanced Save Viewer: Expanded internal save debugging tools to automatically scan active save files. Allows inspection of economy values, building counts, inventories, and progression data without launching gameplay sessions.
