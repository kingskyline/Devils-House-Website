Alpha v0.0005.5 08/06/2026

System Refactor & Performance Upgrade

King Skyline [DHG]

🎮 Player Update

This update was all about making the game smarter behind the scenes — less flashy, but a HUGE improvement for how everything runs.

⸻ Smarter Systems (Big Change)

We’ve completely reworked how the UI and game systems interact:

  • UI no longer controls the game.
  • Buildings now make their own decisions.
  • UI just displays info and sends your input.
  • This makes everything more stable and easier to expand.

⸻ Performance Boost

  • Before: Buttons were checking your money every frame (60x a second).
  • Now: They only update when something actually changes.
  • Result: Smoother performance, especially as the game grows.

⸻ Cleaner UI System

  • All building menus now follow the same structure.
  • Less bugs, less duplication, easier to maintain.
  • This sets up the game for future features without breaking things.

⸻ Better Interaction & Fixes

  • Clicking buildings now consistently opens the correct UI.
  • Everything feels more reliable and predictable.
  • Restored behind-the-scenes logging so we can track economy activity properly.
  • Improved overall system stability.

⸻ What This Means

The game is now more stable, more efficient, and much easier to expand. This is a big groundwork update for future features.

⚙️ Technical Dev Log

Focus: UI decoupling, event-driven optimisation, and architectural cleanup

⸻ MVC-Inspired Refactor

  • Separation of Concerns: Refactored UI architecture to follow Model-View separation principles and removed all game logic from the UI layer.
  • Relocated Logic: Core systems (ResourceProducer, ResourceConverter, Warehouse, SellingStation) now handle logic.
  • Read/Input Layer: UI now acts purely as a read + input layer. It queries state via methods (e.g. affordability checks) and sends actions back to systems.

⸻ Event-Driven Optimisation

  • Eliminated Polling: Removed Update() polling in UI components.
  • Observer Pattern: Implemented using C# Action events.
  • Global Broadcasts: Added OnMoneyChanged (EconomyManager) and OnGlobalStockChanged (Warehouse).
  • Performance Gains: UI now refreshes only when required, reducing CPU usage and improving scalability with large UI counts.

⸻ UI Architecture Standardisation

  • Abstract Base Class: Introduced BaseInspectorUI abstract class to centralise binding and refresh logic.
  • Shared Structure: Removed duplicated inspector code. All inspector panels now inherit from a shared structure for consistent behaviour and easier maintenance.

⸻ Input Integration & Debugging

  • Unified Workflow: Updated ConstructionInput to use a unified .Bind() workflow, ensuring all interactable buildings correctly route to their inspector UI.
  • Logging Restored: Restored production and transaction logging to maintain visibility of economy systems without UI dependency.

⸻ Summary

This update focused on creating a clean separation between UI and game logic, implementing event-driven performance improvements, and establishing a standardised, maintainable UI architecture.
