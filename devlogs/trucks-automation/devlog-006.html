Alpha v0.0006 28/04/2026 - 20:00PM

Economy Overhaul & Smart Systems

King Skyline [DHC]

🎮 Player Update

This is a BIG one — we’ve taken a huge step forward in how the game runs, feels, and scales.

⸻ Massive Performance Upgrade

We completely rebuilt how the UI works:

  • No more constant background checks.
  • Systems now only update when something actually changes.
  • This means smoother performance, even with huge factories.

⸻ New Building: Headquarters

Upgrades are no longer “magic menus”.

  • You now build a Headquarters.
  • Click it to access global upgrades.
  • Makes the game feel more grounded and realistic.

⸻ New Economy System

The old system had a problem: one upgrade could break the entire game. Now:

  • Each resource has its own value upgrades.
  • You choose what to invest in.
  • More strategy, more control.

⸻ Commodities Market

Selling stations are now full Market Boards:

  • Shows all resource values.
  • Lets you upgrade them individually.
  • Updates automatically.
  • Clear, powerful, and easy to use.

⸻ Smarter Core Systems

We built a “global brain” for upgrades:

  • All upgrade logic is now centralised.
  • Buildings no longer store messy data.
  • Cleaner systems, fewer bugs.

⸻ Fixes & Improvements

  • Save system now properly reports what it’s doing.
  • Market transactions are logged clearly.
  • Fixed a weird random system conflict.

⸻ What This Means

The game is now faster, more stable, more strategic, and easier to expand. This is a major step toward a full playable experience.

⚙️ Technical Dev Log

Focus: Event-driven architecture, economy redesign, and system centralisation

⸻ Event-Driven UI Architecture

  • Eliminated Polling: Removed all UI Update() polling across buildings and trucks.
  • Event Broadcasters: Implemented in ResourceProducer, ResourceConverter, Warehouse, and Truck.
  • Targeted Events: Introduced OnStuckStatusChanged and OnStateChanged.
  • Passive Listeners: UI components (BuildingWarningUI, TruckWarningUI) now act as passive listeners.
  • Result: Near-zero CPU usage for UI systems at scale, creating a fully reactive, event-driven feedback loop.

⸻ World-Integrated UI (Headquarters System)

  • Headquarters Building: Removed global floating upgrade menu and introduced Headquarters building for centralised upgrades.
  • Global Upgrades: Truck capacity and export speed are now accessed via world interaction.
  • Result: Stronger player immersion and clearer gameplay structure.

⸻ Economy System Overhaul

  • Scaling Systems: Removed global price multiplier system and implemented per-resource upgrade scaling.
  • Commodities Market: Refactored SellingStation into Commodities Market system. It dynamically generates upgrade UI for all resources, reads directly from save data, and features LINQ-driven dynamic UI population.
  • Result: Prevents late-game economy break and enables targeted player strategy.

⸻ Global System Refactor (UpgradeManager)

  • Centralisation: Centralised upgrade logic into UpgradeManager. Removed local upgrade variables from buildings.
  • Data Querying: Buildings now query global systems for values.
  • Result: Clean data flow, simplified save/load structure, and reduced risk of data inconsistency.

⸻ Debugging & Stability

  • Verbose Logging: Restored SaveManager logging (Version validation, Missing data handling, Load confirmation).
  • Transaction Logging: Added for market system tracking.
  • Conflict Resolution: Resolved Random namespace conflict.

⸻ Summary

This update focused on a full transition to event-driven UI systems, centralised upgrade and economy logic, resulting in improved scalability, performance, and maintainability.
