Save System, Smarter Trucks & Stability

King Skyline [DHC]

🎮 Player Update

Quick update on Trucks & Automation — a lot of behind-the-scenes work has gone in this week to make the game more stable and actually feel like a real, persistent world.

⸻ The Game Now SAVES Everything

Big milestone — the game now remembers your progress including your money, your trucks, your upgrades, and your buildings + their inventories!

  • Press F5 to save.
  • Press F6 to load.
  • Press F7 to wipe everything (for testing).
  • No more restarting from scratch every time!

⸻ We Squashed Some WILD Bugs

Adding a save system broke… a lot. Things are now much more stable.

  • Fixed trucks spawning inside each other and launching into space.
  • Fixed old saves breaking building levels.
  • Fixed systems duplicating and fighting each other.

⸻ Trucks Are WAY Smarter Now

Before, trucks would happily drive into nothing or get stuck forever. Now they feature much more reliable behaviour:

  • They check the road ahead before moving.
  • They stop at dead ends.
  • They won’t drive the wrong way.

⸻ Stability Improvements

  • Cleaner systems behind the scenes.
  • Better error handling.
  • More future-proof setup resulting in less crashing and smoother development going forward.

⸻ What This Means

The game now has a proper save/load system, more stable world systems, and smarter vehicle behaviour. This is a big step toward a fully playable experience.

⚙️ Technical Dev Log

Focus: Persistence, Stability & AI Logic Updates

⸻ Phase 6: Persistence Layer (Save System)

Implemented a structured JSON save/load system (no PlayerPrefs or BinaryFormatter).

  • Core Features: F5 (Save), F6 (Load), F7 (Wipe). Saves player money, fleet size, and upgrade data.
  • World Reconstruction: BuildManager supports data-driven spawning to rebuild the full map automatically on load.
  • Deep Serialization: Captures buildings (inventory + upgrades), trucks (cargo), and producers (stock + progress).

⸻ Phase 7: Stability & Bug Fixes

Resolved edge-case issues from full world rebuilds:

  • Physics Overlap: Fixed trucks colliding on spawn via collider disabling.
  • Legacy Saves: Fixed invalid level 0 states with value clamps.
  • Duplicate Managers: Conflicting instances resolved via singleton enforcement.
  • Inspector Errors: Fixed object deletion issues with safe load and UI reset logic.

⸻ Phase 8: Vehicle Logic Improvements

  • Forward Tile Scanning: Implemented via GridManager.
  • Dead-End Detection: Trucks now stop safely.
  • Direction Validation: Ensured using angle checks.

⸻ Phase 9: Future-Proofing & Key Scripts

  • Updated deprecated APIs to modern Unity equivalents and avoided legacy serialization systems.
  • Added strict null-checks to prevent runtime crashes.
  • SaveManager.cs / GameSaveData.cs: Handles persistence.
  • BuildManager.cs: Handles world reconstruction.
  • Truck.cs: Handles movement logic.
  • ConstructionInput.cs: Handles input/UI stability.
