⚙️ Technical Dev Log

Focus: UI scalability, dynamic data-driven systems, and economy expansion

⸻ Dynamic UI Architecture

  • Data-Driven Prefabs: Replaced hardcoded upgrade UI with a data-driven prefab system.
  • Layout Automation: Implemented reusable UpgradeRowUI using Unity Layout Groups. UI now generates rows based on upgrade lists at runtime.
  • Text Structuring: Added Rich Text hierarchy for improved readability and structure, plus dynamic scrolling panels.
  • Real-Time Validation: Implemented real-time affordability checks. Buttons use System.Func<bool> for continuous validation.

⸻ Economy & Market Systems

  • Instance-Based Upgrades: Refactored Selling Station into an instance-based upgrade system. Added local variables (logisticsLevel, multiplierLevel).
  • Market Board UI: Built Market Board UI to display all item values using dynamic prefab rows.
  • LINQ Sorting: Implemented alphabetical sorting (LINQ OrderBy) applied to Warehouse + Market UI for clarity.

⸻ Input & Interaction

  • Raycasting: Updated raycast handling for new building layers.
  • Panel Management: Expanded ESC key system to support cleanly closing all UI panels.

⸻ Save System Expansion

  • State Migration: Removed deprecated global variables from save data.
  • Extended BuildingSaveData: Now tracks yieldLevel, storageLevel, loadSpeedLevel, logisticsLevel, and multiplierLevel.
  • Backward Compatibility: Added safeguards using fallback values to support older save versions without breaking.

⸻ Summary

This update focused on building scalable, reusable UI systems, expanding economy interactions, and maintaining stability with evolving save data structures.
