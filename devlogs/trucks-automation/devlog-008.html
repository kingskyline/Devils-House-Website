The Economy Overhaul

King Skyline [DHC]

🎮 Player Update

This is a BIG one. We’ve officially moved from “it works” → to a real tycoon-style economy system.

⸻ Smarter, Scaling Economy

Everything now scales properly as you grow, meaning no more easy infinite money:

  • Trucks get more expensive the more you own.
  • Upgrades ramp up FAST in cost.
  • Late game numbers go into the billions!

⸻ Real Production Chains

We’ve balanced the entire system so it feels rewarding:

  • Raw resources: low value (Example: Wood → £3).
  • Processed goods: better.
  • Advanced items: BIG profit (Example: Engine Parts → ~£70).

⸻ Smarter Upgrade System

Upgrades now evolve over time, making you actually feel your factory progressing:

  • Early = basic materials
  • Mid = processed items
  • Late = advanced goods

⸻ Headquarters is now IMPORTANT

We added a real “company brain” building that becomes your late-game power hub:

  • Boosts truck capacity.
  • Improves global export speed.

⸻ Cleaner UI & Dev Improvements

  • Buildings now show clearer production data.
  • Complex recipes display properly and everything updates in real-time.
  • Behind the scenes: Faster icon creation tools, cleaner systems, and more stable code.

⸻ What This Means

This update changes the game from “place stuff and earn money” into “build, scale, optimise, and dominate your economy”. More systems coming next as we keep pushing toward a full tycoon experience!

⚙️ Technical Dev Log

Focus: Economy Architecture, System Standardisation & Scalable Progression

⸻ 1. Centralised Economy Architecture (“Global Brain”)

  • Pricing Logic: Fully migrated into UpgradeManager using a universal formula: BaseCost * (2.5 ^ (Level - 1)).
  • Hardcoding Removed: Removed all hardcoded upgrade arrays across factories, trucks, and market systems.
  • Global Authority: Headquarters.cs now acts as global authority for truck capacity and export speed.
  • Result: Unified pricing logic, consistent scaling across all systems, and clean data flow for save/load.

⸻ 2. Economy & Progression Rebalance

  • Exponential Pricing: Introduced exponential truck pricing: 250 * (1.8 ^ totalTrucksOwned).
  • Baseline Economy: Established with Wood (£3) scaling through a 50% tier profit chain, peaking at Engine Parts (~£70).
  • Phase-Based Requirements: Upgrades require raw materials (early), processed goods (mid), and manufactured items (late) to prevent exploits and ensure long-term scaling.

⸻ 3. Factory System Standardisation

  • Shared Variables: Synced all factory scripts to shared variables (baseCraftTime, speed scaling, logistics speed).
  • Structure Unification: Unified ManufacturingPlant with ResourceConverter structure.
  • Conflict Resolution: Resolved compilation conflicts (CS0102 duplicate definitions).

⸻ 4. Data & UI Integration

  • DataOverlayUI Upgrades: Now supports multi-input recipes and dynamically builds formatted strings via StringBuilder while maintaining camera-facing billboarding.
  • Live System Data: UI now reads directly from live system data (no duplicated logic layer).

⸻ 5. Tooling & Workflow Improvements

Built custom inspector tooling: IconGeneratorEditor with an inline “GENERATE ICON” button. This results in a faster iteration workflow and reduced manual asset setup time.

⸻ 6. Structured Progression Blueprint

Formalised a shopping list progression system for all building tiers by defining:

  • Item pricing tiers and building base costs.
  • Upgrade caps (Buildings: 20 / Trucks: 5).
  • Resource-based upgrade phases.

⸻ Summary

This update marks the transition from a functional prototype to a mathematically structured tycoon system, with fully centralised logic, scalable economy rules, and standardised factory behaviour ready for long-term expansion.
